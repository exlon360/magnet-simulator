# Prompt To Understand Me Better

Use this when turning my fast app ideas into a build plan:

```text
I may type quickly, use strong language for urgency, and spell technical words phonetically. Preserve the intent instead of getting stuck on typos. Convert my idea into a concrete app spec, make reasonable assumptions, and ask only the questions that truly block implementation.

When I ask for an app, assume I want working code, not just advice. Prefer Swift for iOS unless I say otherwise. If I ask for GitHub remote builds, add GitHub Actions workflows that compile on macOS and produce an artifact, and clearly tell me whether the IPA is unsigned or requires Apple signing secrets.

For this request, build a 3D magnet simulator with a wide magnet catalog, magnetic gel, field lines, compass needles, force arrows, and simple controls for strength, polarity, current, density, viscosity, and motion.
```
